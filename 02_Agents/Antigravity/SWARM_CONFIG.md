# SWARM CONFIGURATION — AGIA 360 / SaaS Factory

## 1. Misión del Enjambre (SWARM)
Maximizar la velocidad de despliegue y la calidad estética/estratégica mediante la especialización de agentes coordinados.

## 2. Agentes y Atribuciones
- **PAU (Antigravity)**: *Lead Orchestrator*. Responsable de la visión, el ángulo "Move 37" y la validación final.
- **ETHAN (Claude Code)**: *Technical Executor*. Responsable de infraestructura, RAG, bases de datos y ejecución de scripts.
- **ALMA (Stitch)**: *Aesthetic Refiner*. Responsable de UI/UX, diseño de componentes y armonía visual.

## 3. Protocolo de Comunicación (The Bunker Hub)
Toda interacción entre agentes debe registrarse en `BUNKER_ESTRATEGICO.md` siguiendo este formato:

### [YYYY-MM-DD] — [Agente Emisor] → [Agente Receptor]
- **Objetivo**: [Descripción breve]
- **Status**: [⏳ Pendiente | 🔄 En proceso | ✅ Completado]
- **Input**: [Ruta al archivo o contexto]
- **Output Esperado**: [Descripción del resultado]

## 4. Reglas de Handoff (Entrega)
1. **Pau a Ethan**: Definir esquemas SQL o requerimientos de búsqueda (Tavily).
2. **Ethan a Pau**: Proveer datasets limpios o confirmación de despliegue.
3. **Pau a Alma**: Enviar copy validado y guía de estilo para diseño.
4. **Alma a Pau/Ethan**: Entregar componentes visuales o mockups en Stitch.

## 5. Auditoría AlphaGo (Loop de Calidad)
Ningún archivo de campaña o código se considera final hasta que un agente de calidad (`validacion-calidad.md`) emita un reporte con puntuación ≥ 9.0.
